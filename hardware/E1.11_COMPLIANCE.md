# LuxDMX v5.00 — ANSI E1.11 (DMX512-A) compliance

Checked against **ANSI E1.11-2008 (R2018)**, the current free PDF from
[tsp.esta.org](https://tsp.esta.org/tsp/documents/docs/ANSI-ESTA_E1-11_2008R2018.pdf).

## Is there a certification?

No. Nobody hands out a DMX512-A stamp. Clause 1.6 and the Notice state that compliance is "strictly
voluntary and the responsibility of the manufacturer," and ESTA "does not approve, inspect, or certify"
anything. It is self-declared. The only gate: clause 10 — putting "DMX512-A" on the product takes on its
marking and disclosure duties, which are met below.

## Compliance

LuxDMX v5.00 is a **DMX512-A compliant isolated transmitter**, including the optional **Annex C
"Protected"** overvoltage level. Per isolated universe:

| Clause | Requirement | | Implementation |
|---|---|---|---|
| 4.1 | EIA-485-A electrical | ✅ | ISO3086DWR EIA-485 transceiver |
| 4.2 | Isolation (optional) | ✅ | B0505S isolated DC-DC + ISO3086 barrier (~1 kV), separate VISO/GNDISO island |
| 4.3 / 5.4 | One driver, terminated line, allowed TX topology | ✅ | fully isolated island (Annex A.1 isolated-transmitter) |
| 4.4 / 7.1.1 | Port = data1±, common; data2 NC | ✅ | pin 1 COM, pin 2 D1-, pin 3 D1+, pins 4/5 NC |
| 4.9 | 120 Ω termination | ✅ | 120 Ω across the driver (both line ends terminated for RDM) |
| 5.10 | Drive 32 unit loads | ✅ | ISO3086 is 1/8 unit load |
| 6.2 | ESD 4 kV contact / 8 kV air | ✅ | SM712 TVS at each XLR (±15 kV IEC 61000-4-2) |
| 6.1 / Annex C | **Protected** — survive 30 VAC / 42 VDC on the pins | ✅ | series Bourns TBU-CA065-200-WH per data line + SM712: the TBU triggers in <1 µs and blocks the fault (650 V standoff), the SM712 takes the sub-µs transient. 4 mm creepage to board GND |
| 7.1 | Female XLR on a transmitter | ✅ | Neutrik NC5FAH 5-pin XLR, female |
| 8 / 9 | Break, MAB, 250 k, 2 stop bits, NULL start, framing | ✅ | firmware (esp_dmx) |
| 10.2 | Port marking, Table 8 abbreviations | ✅ | silk: COM / D1- / D1+ |
| 10.4 | Mark the isolated port | ✅ | ISO on the XLR silk |
| 10.5.3 / .4 / .5 | Manual: loss-of-data, latency, NULL-start range | ✅ | README: holds last frame; ~1 refresh + frame time; NULL start 0–255 |

A fail-safe bias (~470 Ω, A→VISO / B→GND on the isolated side) holds a defined idle MARK across the RDM
turnaround. Not an E1.11 requirement — an RDM-receive robustness add.
