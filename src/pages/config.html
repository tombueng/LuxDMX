DMX Output / RDM Pins
The numbers below are ESP32 GPIO numbers — the GPIO xx / IO xx / Gxx labels printed on your board next to each pin (the chip pin, not the physical header position). Wire each one to the matching pin on your RS485 transceiver. Transceiver boards name their pins inconsistently, so match by function — common aliases are listed per field. (The RS485 side A/B and VCC/GND are wired separately — see below.)
Which ESP32 hardware serial peripheral drives the line. UART1 is fine on all boards.
ESP32 GPIO that sends DMX. Wire to the transceiver's driver input — labeled DI (or TXD/TX; on some cheap modules, confusingly, RXD).
ESP32 GPIO that receives fixture replies (needed for RDM). Wire to the transceiver's receiver output — labeled RO (or RXD/RX). If RDM/output won't work, try swapping TX ↔ RX.
ESP32 GPIO that switches the transceiver between transmit and receive. Wire to its direction pin — labeled EN, or DE and RE tied together. Set this to enable RDM (auto-discovery + remote addressing). −1 disables RDM and is required for auto-direction modules (no EN/DE-RE pin, e.g. the Waveshare C). Applied after reboot.
Not set here (wire directly): the transceiver's RS485 side A/B (a.k.a. D+/D−) → XLR pin 3 / pin 2, GND → XLR pin 1; and its VCC→3.3 V, GND→GND. Add a 120 Ω terminator across A/B at the far end of the DMX chain (required for RDM).
Example (ESP32-S3 + MAX3485 "EN" board): TX=17→DI, RX=16→RO, RTS=4→EN.
